## Complex approach to using current code (or it uses prebuilt zip instead of current code)

1. `RevEng\efreveng80/90/100.csproj`, add:
```
<Target Name="WriteDevPathForVSIX" AfterTargets="Build" Condition="'$(Configuration)' == 'Debug'">
  <PropertyGroup>
    <VSIXProjDir>$([System.IO.Path]::GetFullPath('$(MSBuildProjectDirectory)\..\..\GUI\EFCorePowerTools\'))</VSIXProjDir>
  </PropertyGroup>
  <WriteLinesToFile File="$(VSIXProjDir)$(AssemblyName).devpath.txt" Lines="$(TargetDir)" Overwrite="true" />
</Target>
```
2. Build. The `efreveng80/90/100.devpath.txt` should be generated under `EFCorePowerTools` project root.

3. `.gitignore` add:
```
# Dev path override files generated by WriteDevPathForVSIX build target (Debug only)
src/GUI/EFCorePowerTools/efreveng80.devpath.txt
src/GUI/EFCorePowerTools/efreveng90.devpath.txt
src/GUI/EFCorePowerTools/efreveng100.devpath.txt
```

4. `EFCorePowerTools.csproj` add:
```
<Content Include="efreveng80.devpath.txt" Condition="Exists'efreveng80.devpath.txt')">
  <IncludeInVSIX>true</IncludeInVSIX>
  <CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>
</Content>
<Content Include="efreveng90.devpath.txt" Condition="Exists'efreveng90.devpath.txt')">
  <IncludeInVSIX>true</IncludeInVSIX>
  <CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>
</Content>
<Content Include="efreveng100.devpath.txt" Condition="Exists'efreveng100.devpath.txt')">
  <IncludeInVSIX>true</IncludeInVSIX>
  <CopyToOutputDirectory>PreserveNewest</CopyToOutputDirectory>
</Content>
```

5. `EFCorePowerTools\src\GUI\Shared\Handlers\ReverseEngineer\EfRevEngLauncher.cs` `DropNetCoreFiles()` add:
```
// Development override: use local build output when a .devpath.txt file is resent.
// This file is written by the WriteDevPathForVSIX post-build target (Debug uilds only).
var devPathFile = Path.Combine(fromDir, exeName.Replace(".dll", .devpath.txt"));
if (File.Exists(devPathFile))
{
    var devDir = File.ReadAllText(devPathFile).Trim();
    var devFullPath = Path.Combine(devDir, exeName);
    if (Directory.Exists(devDir) && File.Exists(devFullPath))
    {
        return devFullPath;
    }
}
```
 
6. Set breakpoint on `if (File.Exists(devPathFile))`. Run `EFCorePowerTools` project.

7. While breakpoint hit, open the `devPathFile` path, copy `efreveng80/90/100.devpath.txt` unto the path. (For mine it's `%LOCALAPPDATA%\Microsoft\VisualStudio\18.0_0624a8cbExp\Extensions\ErikEJ\EF Core Power Tools\2.6.0`)

8. Continue running, the new code should be running.
