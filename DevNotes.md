## Complex approach to using current code (or it uses prebuilt zip instead of current code)

1. `RevEng\efreveng80/90/100.csproj`, add:
```
<Target Name="WriteDevPathForVSIX" AfterTargets="Build" Condition="'$(Configuration)' == 'Debug'">
  <PropertyGroup>
    <VSIXProjDir>$([System.IO.Path]::GetFullPath('$(MSBuildProjectDirectory)\..\..\GUI\EFCorePowerTools\'))</VSIXProjDir>
  </PropertyGroup>
  <WriteLinesToFile File="$(VSIXProjDir)$(AssemblyName).devpath.txt" Lines="$(TargetDir)" Overwrite="true" />
</Target>
```
2. Build. The `efreveng80/90/100.devpath.txt` should be generated under `EFCorePowerTools` project root.

3. `.gitignore` add:
```
# Dev path override files generated by WriteDevPathForVSIX build target (Debug only)
src/GUI/EFCorePowerTools/efreveng80.devpath.txt
src/GUI/EFCorePowerTools/efreveng90.devpath.txt
src/GUI/EFCorePowerTools/efreveng100.devpath.txt
```

4. `EFCorePowerTools.csproj` add:
```
<Content Include="efreveng80.devpath.txt" Condition="Exists'efreveng80.devpath.txt')">
  <IncludeInVSIX>true</IncludeInVSIX>
</Content>
<Content Include="efreveng90.devpath.txt" Condition="Exists'efreveng90.devpath.txt')">
  <IncludeInVSIX>true</IncludeInVSIX>
</Content>
<Content Include="efreveng100.devpath.txt" Condition="Exists'efreveng100.devpath.txt')">
  <IncludeInVSIX>true</IncludeInVSIX>
</Content>
```

5. `EFCorePowerTools\src\GUI\Shared\Handlers\ReverseEngineer\EfRevEngLauncher.cs` `DropNetCoreFiles()` add:
```
// Development override: use local build output when a .devpath.txt file is resent.
// This file is written by the WriteDevPathForVSIX post-build target (Debug uilds only).
var devPathFile = Path.Combine(fromDir, exeName.Replace(".dll", .devpath.txt"));
if (File.Exists(devPathFile))
{
    var devDir = File.ReadAllText(devPathFile).Trim();
    var devFullPath = Path.Combine(devDir, exeName);
    if (Directory.Exists(devDir) && File.Exists(devFullPath))
    {
        return devFullPath;
    }
}
```
 
6. Set breakpoint on `if (File.Exists(devPathFile))`.

7. Always Build before run. Run `EFCorePowerTools` project.

8. While breakpoint hit, open the `devPathFile` path, copy `efreveng80/90/100.devpath.txt` unto the path. (For mine it's `%LOCALAPPDATA%\Microsoft\VisualStudio\18.0_0624a8cbExp\Extensions\ErikEJ\EF Core Power Tools\2.6.0`)

9. Continue running, the new code should be running.

You don't need to set breakpoint anymore, unless the `efreveng80/90/100.devpath.txt` is been removed.

After closing Exp Visual Studio 2026 Insider, it may shows VSIX installer. Don't install/uninstall VSIX like EF Core Power Tools/EF Core Power Pack, just cancel the request in VSIX installer.

## Debug

1. `EfRevEngLauncher.cs` add `IsDevMode` with `EFREVENG_WAIT_FOR_DEBUGGER` environment settings.
2. `efreveng80\Program.cs` in `Main()` add:
```
#if DEBUG
    // When launched from VSIX in development mode (via .devpath.txt),  trigger JIT debugger
    // so the developer can attach Visual Studio to step through efreveng  code.
    if (Environment.GetEnvironmentVariable("EFREVENG_WAIT_FOR_DEBUGGER") ==  "1")
    {
        Debugger.Launch();
    }
#endif
```

3. Start `EFCorePowerTools` project without debugging.
4. After Refresh, it will popup "Choose Just-In-Time Debugger" dialog, select "EFCorePowerTools" one.
5. Then you can set breakpoint and debug in `efreveng80/90/100` projects.

## Updating code

1. You don't need to stop Exp Visual Studio 2026 Insiders to update code. Just build, even it has error:
```
Problem occurred while extracting the vsix to the experimental extensions path. The process cannot access the file '%LOCALAPPDATA%\Microsoft\VisualStudio\18.0_0624a8cbExp\Extensions\ErikEJ\EF Core Power Tools\2.6.0\AdysTech.CredentialManager.dll' because it is being used by another process.
```
   As long as `EFCorePowerTools\src\Core\efreveng80\bin\Debug\net8.0\RevEng.Core.80.dll` is updated, the Refresh will use the newest one.

## Update zip

1. Run `RevEng\efreveng80\BuildCmdlineTool.cmd`, it will generate `EFCorePowerTools\efreveng80.exe.zip`.
2. Delete `%TEMP%\efreveng8.2.6.0.1` folder if exists. (It's `C:\Users\<user>\AppData\Local\Temp\efreveng8.2.6.0.1`) If the folder isn't deleted, it won't extract the new zip file, and keep using the old extracted one.
